# 우리 가족 맞춤 복지 찾기 (정적 사이트)

상황(영유아·임신출산·한부모·저소득 등)을 **켜기만 하면** 받을 수 있는 복지 지원을 빠짐없이 펼쳐 주는 **GitHub Pages용 정적 사이트**입니다. 검색이 아니라 *조건이 자격을 정하는* 방식이며, 신청에 필요한 서류를 통합 집계해 "서류 N종만 준비하면 M개 신청" 형태의 행동 가능한 결과를 제공합니다.

- **빌드 도구 0** — `index.html` 단일 파일 + 바닐라 JS. 그대로 GitHub Pages에 올리면 동작합니다.
- 로직은 순수 함수(`window.WF = {collect, matchFull, bulkDocs, readyCheck}`)로 분리되어 node로 테스트 가능합니다.

## 구성

| 파일 | 설명 |
|---|---|
| `index.html` | 앱 본체 (상황 칩 → 결과 2등급: ⭐큐레이션 / 🔄복지로 전체) |
| `tokens.css` | 디자인 토큰 (`.pb-` 클래스, 플레이북 공통) |
| `data/welfare.json` | ⭐ **검증 큐레이션 15건** — 사람이 공식 출처를 직접 확인해 정리 (서류·기한·신청 방법 포함) |
| `data/welfare_full.json` | 🔄 **복지로 공개 목록 5,012건** (중앙부처 451 + 지자체 4,561, 약 3.3MB) |

## 데이터 출처와 신뢰 등급

- 출처: [복지로](https://www.bokjiro.go.kr) 공개 복지서비스 목록
- **수집일: 2026-06-11** (이후 변경분은 반영되지 않음)
- 태그 분류: **86%는 복지로 공식 분류**, 나머지 14%는 키워드 자동분류(`confidence` 필드에 명시)
- 결과는 신뢰 2등급으로 구분 표시합니다:
  - ⭐ 검증 큐레이션(15건): 사람이 공식 출처 확인 — 서류·택1 분기(가정양육수당↔보육료)·소득 기준 필터까지 지원
  - 🔄 복지로 전체(5,012건): 공개 데이터를 그대로 비추는 거울 — 이름·요약·기관·복지로 상세 링크만 제공

### 재수집 방법

`welfare_full.json`은 복지로 공개 목록을 스크립트로 수집한 스냅샷입니다. 갱신하려면 원 수집 스크립트(별도 리포 `playbook-dryrun-welfare-fasthtml`의 데이터 파이프라인)를 다시 실행해 `data/welfare_full.json`을 교체하고, `index.html`·이 문서의 수집일을 갱신하세요.

## 로컬 실행

정적 파일이지만 데이터를 `fetch`로 읽으므로 **http로 서빙**해야 합니다 (file:// 직접 열기 불가).

```bash
python -m http.server 5041
# http://localhost:5041
```

로직 자가 테스트(선택): `index.html`의 `WF-LOGIC` 블록을 추출해 node로 실행하는 방식이며, 검증 시나리오는 "영유아(영아·영유아, 가정양육)+임신·출산+한부모+중위 50%+서울 강남구 → 큐레이션 8건·서류 9종, 보육료 제외"입니다.
